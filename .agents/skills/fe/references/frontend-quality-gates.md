# FE 품질 게이트

## 1. 웹표준과 시맨틱 마크업

- 레이아웃은 가능한 한 `header`, `nav`, `main`, `section`, `article`, `aside`, `footer` 같은 landmark를 우선 사용한다;
- 제목은 시각 크기가 아니라 `h1`~`h6` 계층으로 잡고, 한 화면에 `h1` 목적이 불분명하면 이유를 설명한다;
- 클릭 동작은 `button`, 이동 동작은 `a`를 사용하고, `div` + `onClick` 조합은 대체 수단이 없는 경우만 허용한다;
- 표 데이터는 `table` 계열 태그를 유지하고, 시각 배치 목적이면 `div` 레이아웃으로 푼다;

## 2. 접근성

- 모든 인터랙티브 요소는 키보드만으로 도달 가능해야 한다;
- 포커스 표시를 제거하지 않고, 커스텀 스타일이 필요하면 `:focus-visible` 대안을 제공한다;
- 아이콘 버튼, 닫기 버튼, 토글 버튼은 눈에 보이는 텍스트가 없으면 접근 가능한 이름을 반드시 제공한다;
- `ARIA`는 네이티브 요소로 해결되지 않을 때만 최소한으로 사용한다;
- 상태 전달은 색상만으로 하지 않고 텍스트, 아이콘, 설명 문구 중 하나를 추가한다;

## 3. 폼과 유효성검사

- `label`과 입력 요소를 연결하고, 도움말과 오류 문구는 `aria-describedby`로 연결한다;
- 필수 입력은 `required` 여부와 시각 표시를 함께 제공하고, 서버 검증 실패 시에도 같은 오류 영역을 재사용한다;
- placeholder를 라벨 대체재로 쓰지 않는다;
- 숫자, 이메일, 날짜, 자동완성은 가능한 한 적절한 `type`, `inputMode`, `autoComplete`를 지정한다;

## 4. 크로스브라우징과 하위 호환성

- 최신 브라우저 전용 API는 지원 범위를 확인하고 도입한다;
- 대체 가능한 기본 HTML 동작이 있으면 JavaScript 의존 구현보다 우선한다;
- 애니메이션, `backdrop-filter`, `dialog`, Clipboard API, View Transition API 같은 기능은 브라우저 제약을 먼저 점검한다;
- 기능이 빠져도 핵심 흐름이 막히지 않도록 progressive enhancement를 기본 전략으로 둔다;

## 5. 검증

- 자동 검증: `eslint`, 타입 체크, 빌드;
- 수동 검증: 탭 이동, `Shift+Tab`, `Enter`, `Space`, `Escape`, 스크린리더 이름 추정, 모바일 뷰포트, 주요 브라우저 렌더링 확인;
- lint는 구조적 위험을 줄이는 도구이지, 실제 사용성이나 문맥 적합성 자체를 보장하지 않는다;
