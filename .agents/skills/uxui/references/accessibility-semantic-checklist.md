# UXUI 접근성·시맨틱 체크리스트

## 1. 정보 구조

- 페이지마다 핵심 목적과 주요 과업을 한 줄로 설명할 수 있어야 한다;
- 제목 계층은 정보 구조와 일치해야 하고, 스타일만으로 제목 역할을 대체하지 않는다;
- 카드 목록, 탭, 표, 탐색 메뉴는 시각 모양이 아니라 데이터 구조에 맞는 패턴을 선택한다;

## 2. 인터랙션

- 열림/닫힘, 선택/해제, 진행 중, 완료, 오류를 사용자에게 즉시 알려야 한다;
- 모달, 드로어, 드롭다운은 열렸을 때 첫 포커스와 닫힐 때 반환 포커스를 설계한다;
- hover 전용 인터랙션에 의존하지 않고 터치와 키보드 흐름을 함께 설계한다;

## 3. 폼과 오류

- 라벨, 도움말, 예시값, 오류 메시지의 우선순위를 분리해서 설계한다;
- 오류는 문제 원인과 수정 행동을 함께 안내하고, 색상만으로 전달하지 않는다;
- 실시간 검증이 필요하면 입력 중 방해를 최소화하고, 제출 시점 검증과 충돌하지 않게 정한다;

## 4. 시각 대비와 상태 표현

- 선택, 강조, 성공, 경고, 오류 상태는 색상 외 보조 수단을 함께 둔다;
- 텍스트 대비와 비텍스트 대비를 함께 고려하고, disabled 상태도 읽을 수 있어야 한다;
- skeleton, empty, loading 상태도 읽기 순서와 의미를 설명할 수 있어야 한다;

## 5. 브라우저와 하위 호환성

- 새 CSS/브라우저 API를 쓸 때는 지원 범위와 대체 표현을 먼저 정한다;
- 스크롤, 고정 헤더, 입력 포커스, 키보드 오버레이, 모바일 safe area 같은 환경 차이를 고려한다;
- 장식 효과가 사라져도 핵심 정보와 행동 경로는 유지되어야 한다;
