# 자산 부채 파싱 초기 매칭 완화와 편집 재매칭 분리

## 변경 내용

- 자산/부채 파싱 결과 시트의 초기 자동 매칭은 `이름 + 유형`까지 허용해 기존 자산/부채가 다시 업데이트 대상으로 잡히도록 조정한다.
- 사용자가 이름, 유형, 세부 유형을 편집한 뒤에는 정확 일치(`이름 + 유형 + 세부 유형`)만 재매칭으로 인정한다.
- 매칭 키를 바꾸지 않은 편집(예: 금액 수정)에서는 기존 느슨한 매칭 상태를 유지해 의도치 않은 매칭 해제를 막는다.

## 결정 사항

- 초기 진입과 사용자 편집 이후의 요구사항이 다르므로, 하나의 매칭 규칙으로 통일하지 않고 단계별 규칙을 분리한다.
- 사용자가 매칭값을 직접 바꾼 뒤에는 부분 일치로 기존 계정을 유지하지 않고, 정확한 대상이 있을 때만 다시 업데이트 상태로 전환한다.

## 다음 할 일

- 기존 자산/부채가 있는 상태에서 초기 자동 매칭, 세부 유형 변경 후 매칭 해제, 다시 정확한 값으로 맞췄을 때 재매칭되는지 수동 점검한다.
