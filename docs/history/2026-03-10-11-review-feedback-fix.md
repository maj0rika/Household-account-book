# 코드리뷰 피드백 반영

## 변경 내용

- 자산/부채 파싱 결과 시트의 각 항목이 자기 자신의 초기 파싱값을 함께 보존하도록 바꿔, 삭제 후 인덱스가 바뀌어도 초기 매칭 복원 기준이 흔들리지 않게 수정한다.
- 인증 세션 헬퍼에서 Better Auth의 복구 가능한 세션 오류만 `null`로 처리하고, 실제 세션 조회 실패는 그대로 다시 던지도록 예외 범위를 축소한다.

## 변경된 파일

- docs/history/2026-03-10-11-review-feedback-fix.md
- docs/implementation-plan.md
- src/components/assets/AccountParseResultSheet.tsx
- src/server/auth.ts

## 결정 사항

- 재매칭 복원 기준은 리스트 인덱스가 아니라 각 항목의 불변 초기값에 묶어야 삭제/재정렬 이후에도 일관된 동작을 보장할 수 있다.
- 인증 인프라 장애와 단순 세션 만료를 같은 `null`로 취급하면 서버 장애를 숨기게 되므로, `INVALID_TOKEN` 계열만 복구 대상으로 제한한다.

## 다음 할 일

- 자산/부채 파싱 시트에서 삭제 후 남은 항목을 편집해도 초기 매칭 복원과 배지 전환이 유지되는지 수동 점검한다.
- 만료 쿠키와 실제 DB 장애를 각각 재현해 로그인 리다이렉트와 서버 에러 노출이 의도대로 분리되는지 확인한다.
