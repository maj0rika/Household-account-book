# 자산 부채 파싱 초기 매칭 복원 경로 추가

## 변경 내용

- 자산/부채 파싱 결과 시트에서 사용자가 편집 중 값을 바꿨다가 초기 파싱 시점의 매칭 아이덴티티로 되돌리면, 초기 자동 매칭도 다시 `업데이트` 상태로 복원되게 수정한다.
- 편집 후 정확 일치 매칭과 별개로, 초기 느슨한 매칭 상태를 되돌릴 수 있는 복구 경로를 추가한다.

## 변경된 파일

- docs/history/2026-03-10-10-account-parse-restore-initial-match.md
- docs/implementation-plan.md
- src/components/assets/AccountParseResultSheet.tsx

## 결정 사항

- 초기 자동 매칭과 편집 후 정확 재매칭은 규칙이 다르므로, 편집 중 되돌리기 UX를 위해 초기 매칭 복원 경로를 별도로 둔다.
- 이 복원은 현재 값이 초기 파싱 아이덴티티와 정확히 같을 때만 허용한다.

## 다음 할 일

- 초기 자동 매칭된 항목에서 세부 유형을 바꿨다가 다시 초기 값으로 되돌렸을 때 `업데이트` 배지가 복원되는지 수동 점검한다.
