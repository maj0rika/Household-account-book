# 2차 디자인 시스템 전수검사 리뷰 완료

## 요청 요약

디자인 시스템 전수검사 리뷰 (추가 컴포넌트 터치 영역 확보)

## 변경 내용

### 1. 모바일 렌더링 최적화 및 접근성 개선 (UI 컴포넌트)

- Select 컴포넌트 모바일 터치 해상도(최소 44px) 대응을 위해 기본 사이즈 상향 (`h-9` -> `h-11`)
- SelectItem 모바일 터치 해상도를 위해 최소 높이 할당 (`min-h-11`)
- Switch 컴포넌트 모바일 컨트롤 대응 위해 기본 사이즈 상향 (`h-[1.15rem] w-8` -> `h-6 w-11`)
- Switch 썸네일 크기 보정 (`size-4` -> `size-5`, Translate `[calc(100%-2px)]` -> `5`) 및 `border-2` 도입으로 안정적인 토글 UI 확보

## 변경된 파일

| 파일                           | 작업 | 설명                                             |
| ------------------------------ | ---- | ------------------------------------------------ |
| `src/components/ui/select.tsx` | 수정 | `size.default` 토큰과 Item 요소 44px 사이즈 반영 |
| `src/components/ui/switch.tsx` | 수정 | 루트 h-6 w-11 사이즈, 썸네일 20px 사이즈로 확장  |

## 설계 결정

### 핵심 폼 요소 사용성 및 시각적 안정감 강화

- **선택**: `Select`, `Switch` 컴포넌트 등의 기본 크기를 상향(Tailwind `h-11`, `h-6`)하고, 내부 요소 크기도 함께 조정했습니다.
- **이유**: 클릭 오작동 등 폼 입력 과정에서의 허들을 줄이고 접근성을 향상시키기 위해 표준화시켰습니다.
- **대안 (미채택)**: 기존 사이즈 유지 및 패딩/마진 통한 우회 처리. (구조적 통일성을 해칠 우려가 있어 직접 크기를 조절했습니다.)
