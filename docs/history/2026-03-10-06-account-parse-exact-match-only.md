# 자산 부채 파싱 정확 일치 매칭으로 조정

## 변경 내용

- 자산/부채 파싱 결과 시트의 기존 계정 매칭 기준을 이름, 유형, 세부 유형의 정확 일치로 제한한다.
- 편집 중 사용자가 매칭값을 바꾸어 정확 일치가 깨지면 기존 계정 매칭을 해제하고 `신규` 상태로 전환한다.
- 편집 후 다른 기존 계정과 정확히 일치하는 경우에만 업데이트 대상으로 다시 연결한다.

## 변경된 파일

- docs/history/2026-03-10-06-account-parse-exact-match-only.md
- docs/implementation-plan.md
- src/components/assets/AccountParseResultSheet.tsx

## 결정 사항

- 부분 일치(`이름 + 유형`, `이름만`)는 사용자가 매칭을 깨뜨린 상황에서도 기존 업데이트 대상으로 남는 부작용이 더 커서 제거한다.
- 파싱 시트의 업데이트 기본 동작은 "정확히 같은 기존 계정"일 때만 유지한다.

## 다음 할 일

- 자산/부채 파싱 결과 시트에서 이름, 유형, 세부 유형을 각각 바꿔가며 매칭 해제와 재매칭이 기대대로 보이는지 수동 점검한다.
