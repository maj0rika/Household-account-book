# Sub-agent Delivery Pipeline (PM → FE/BE → Infra → Reviewer → Approval)

이 문서는 가계부 프로젝트 변경 작업을 일관되게 처리하기 위한 운영 절차입니다.

## 단계

1. **PM**
   - 이슈 요약, 원인 가설, 범위/수용 기준 정의
2. **FE/BE**
   - 기능 구현 + 동시성/동기화 안전성 반영
   - `npm run lint`, `npm test`, `npx tsc --noEmit` 통과
3. **Infra**
   - 배포/롤백/환경 영향 점검
4. **Reviewer**
   - 수용 기준 충족 여부 + 잔여 리스크 검토
5. **사용자 승인 게이트 (필수)**
   - 변경 요약 공유 후 `커밋/푸시 진행할까요?` 명시적 승인 요청
6. **Commit/Push**
   - 승인 후에만 수행

## 커밋 메시지 규칙

- `fix(accountbook): <요약>`
- 필요 시 본문에 `-` 목록으로 핵심 변경점 기록

## 실패 시 처리

- 어느 단계든 실패 시 즉시 중단
- 실패 원인 / 영향 / 다음 액션을 묶어서 사용자에게 보고
